    브랜치 온보딩 가이드¶

    • 작성자 : 손보형
    • 업데이트: 2019년 9월 29일

    개요¶

    이 문서는 Branch 온보딩시 알고 계셔야하는 주요 사항들 전반적인 진행순서와 함께 설명한 문서입니다. 이 문서에서 설명된 내용 이외 추가로 궁금하신 사항들은 담당 Client Success Manager 또는 Solutions Engineer에게 문의 주시기 바랍니다.

    담당자별 업무소개¶

    • Account Executive (AE): 신규 고객에 대한 세일즈 및 계약을 담당 합니다.
    • Client Success Manager (CSM) 계약 후 대쉬보드 사용법 및 설정 및 Branch를 통한 전반적인 마케팅 집행에 대한 가이드를 드리며 재계약을 담당합니다.
    • Solutions Engineer: Branch SDK 및 API 연동에 대한 가이드 기술지원을 담당합니다.
    • Support Engineer: Branch에 관련된 모든 문의및 이슈를 처리 해드리며, 메일 전송시 support@branch.io를 To로 해서 보내주시기 바랍니다. 필요한 경우 CSM또는 Solutions Engineer를 CC에 추가해서 메일을 보내 주셔도 무방합니다. 한국 업무시간대에는 한국인 엔지니어 분들이 답변을 드립니다.

    온보딩 순서¶

    신규앱 생성¶

    • 기존에는 각각의 앱이 별도로 존재 했으나, 지금은 하나의 Organization 하위에 복수의 앱이 포함되도록 설정할 수 있습니다.
    • 복수의 앱이 있거나 있을 것으로 예상되시는 경우 Organization을 생성하고 그 아래에 앱들을 등록 해야 하므로 담당 CSM에게 해당 내용을 알려주시기 바랍니다.

    • Organization이 생성된 상태에서 신규앱을 생성하시는 경우, 대쉬보드 좌측상단의 건물그림이 있는 Organization페이지에서 해당 이름을 클릭할때 좌측 하단에 표시되는 [Create New App]버튼을 통해서 앱을 생성하셔야 해당 Organization 하위로 신규앱이 생성되며, 특정앱의 페이지에서 신규앱을 생성하시는 경우 Organization하위가 아닌 별도의 앱으로 생성되므로 주의해 주시기 바랍니다.

      하나의 Organization 아래에 복수의 앱이 존재하는 경우, 추후 제공될 Cross-platform ID의 경우 서로 다른앱을 설치해도 동일한 기기인 경우 동일한 Cross-platform ID가 설정됩니다.

    • 앱정보 설정

      • 앱 생성후 Link Settings 페이지에서 설정합니다.
      • iOS:
        • Universal Links : 유니버셜 링크는 Branch의 강점중 하나입니다. 디폴트로 활성화 해주시기 바랍니다.
        • URI Scheme: 앱의 딥링크 스킴 설정.
        • AppStore URL: 앱스토어 랜딩 페이지 URL. (앱이 라이브된 경우 검색을 통한 설정 가능)
        • Bundle Identifier: 애플 개발자 계정에서 Bundle ID를 확인해서 설정합니다. 가이드
        • Apple App Prefix: 애플 개발자 계정에서 Apple App Prefix를 확인해서 설정합니다. 가이드
      • Android:
        • App Links : (선택사항) 앱에서 지원하는 경우에 한해서 활성화 합니다.
        • URI Scheme: 앱의 딥링크 스킴 설정.
        • Google Play URL: 앱스토어 랜딩 페이지 URL.(앱이 라이브된 경우 검색을 통한 설정 가능)
        • Package Name: 앱의 패키지 네임은 Google Play URL에서 확인 가능합니다. id로 값이 설정된 부분.

    • SDK 연동시 필요한 Branch Key와 Branch Secret은 [SETUP & TESTING] - [Account Settings] 페이지의 Profile탭에서 확인할 수 있습니다.

      Branch Key와 Branch Secret은 외부에 유출이 되지 않도록 유의해 주시기 바랍니다.

    • Profile탭 하단의 타임존은 Facebook등의 매체와의 데이터 매칭시 사용되므로 적절하게 설정하시기 바라며 한번 설정후 변경하지 않으시는것을 권장합니다.

    Persona 임포트¶

    • Branch를 사용하시기 이전에 이미 앱이 마켓에 릴리즈되어 이미 많은 인스톨이 발생되어 있는 경우에만 필요하며, 해당 기기들에 대한 광고아이디를 보유하고 계셔야 합니다.
    • 이미 인스톨한 기기들의 광고아이디를 확보하지 못한 경우 Branch SDK를 연동한 앱을 미리 라이브 시키고 인스톨 추이를 확인하시면서 기존 인스톨 사용자들의 앱업데이트 추이를 대략적으로 확인할 수 있습니다. Universal Ads를 사용하시는 고객분들은 기존 인스톨했던 유저들로 부터 인스톨이 대부분 발생한 것을 확인 후 광고캠페인을 집행 하시기 바랍니다.
    • Persona임포트는 이미 인스톨이 발생한 기기의 광고아이디및 추가정보들을 시스템에 임포트 하는 작업이며, 이후 Branch SDK가 연동된 앱이 릴리즈된 후 이미 임포트된 기기인 경우 해당 릴리즈로 사용자가 앱업데이트를 할때 인스톨이 발생하지 않게 됩니다.
    • CSM팀 또는 서포트팀(support@branch.io)으로 임포트할 데이터를 전달해 주시기 바랍니다.

    • 자세한 사항은 아래의 문서를 참고해주시기 바랍니다.

      • Importing Historical User Data

      Persona임포트를 하기 위해서는 Branch SDK가 연동된 앱이 라이브되기 최소 2주 이전에 데이터를 전달 해 주시기 바랍니다.

    트래킹할 이벤트 선정 및 리뷰¶

    • 마케팅팀과 개발팀이 함께 진행하셔야 하는 작업입니다.
    • 마케팅팀은 마케팅효율을 측정하기 위한 이벤트를 선정하셔야하며, 개발팀에서도 트래킹이 필요한 이벤트들을 추가 하실 수 있습니다.

    • 이벤트에 대해서 Solutions Engineer와 함께 리뷰후 실제 개발 작업을 해주시는 것이 권장됩니다.

      개인정보에 관련된 정보들은 가급적 설정하지 않으시는것을 권장드리며, 개인정보로 판단될수 있는 정보를 이벤트에 설정시 사내 개인정보 관련부서의 리뷰를 받으신후 진행하시기 바랍니다.

      개발팀에서는 이벤트연동시 사내 개인정보관련팀에서 검토후 명확히 설정을 요청받은 정보들만 설정하시고, 요청되지 않은 정보를 개인적인 판단으로 설정하시는것은 지양해 주시기 바랍니다.

    • Branch에서 수집하는 정보들과 개인정보 정책은 아래의 문서를 참고 해주시기 바랍니다.

    • SDK Privacy Control

      • SDK Privacy Controls 가이드
      • Branch는 고객사의 개인정보를 매우 중요하게 여기며 특정 데이터수집기능을 비활성화하는 기능을 제공하고 있습니다. Branch의 서비스를 통해서 수집되는 데이터에 대해서는 Privacy Policy 페이지에 자세한 내용들이 설명되어 있습니다.
      • Branch는 딥링킹, 애트리뷰션, 분석 서비스를 위해서 제한적인 기기정보를 수집하고 있습니다만 일부 엔드유저들은 이 Data Processing에서 제외될 필요가 있다는 것과 또는 법(또는 Branch정책)에 의해서 특정 엔드유저(예를틀면, 13세 미만의 아동)들의 특정 타입의 개인정보가 Branch로 제공되는것을 허용하지 않는 경우들이 있다는 것을 알고 있습니다. 이러한 이유를로 인해서 고객사에 SDK Privacy Control 기능을 제공하고 있습니다.
      • SDK Privacy Control기능을 통해서, Branch가 연동된 앱 또는 웹사이트을 가진 고객사에서는 특정 엔드유저의 개인정보 Processing 중지를 요청할 수 있습니다. SDK Privacy Control기능이 특정 엔드유저에 대해 활성화 되었을때도 해당유저는 Branch Link의 생성 및 공유가 가능하며 기본적인 딥링킹 기능은 계속 동작할 것입니다. 다만, 이후의 모든 행동들에 대해서 브랜치로는 해당 엔드유저를 특정할수 있는 정보들이 제외되므로, 해당 엔드유저들에 대해서는 Branch에서 크로스플랫폼 데이터를 통해 제공하는 매끄러운 고객의 여정 서비스를 제공받지 못하게 됩니다.
      • SDK Privacy Control 기능 구현 관련
        • Do Not Track Mode 활성화 구현 예
          • 사용자가 요청시 원격으로 해당 기기에 대해서 활성화 시키도록 구현
          • 앱내 설정 메뉴에서 사용자가 직접 활성화 시킬 수 있는 메뉴를 추가
          • 사용자 인증단계에서 미성년인것이 확인된 경우 자동 활성화
      • 사용자 트래킹 Opt-out
        • User Tracking Opt-out 페이지를 통해서 사용자는 직접 자신을 트래킹에서 제외할 수 있으며, 자세한 사항은 이 페이지의 내용을 참고해 주시기 바랍니다.
        • 사용자가 스스로 Opt-out을 할 필요가 있을 경우 해당 페이지를 노출시켜 주시면 됩니다.
        • Branch에서 제공하는 Opt-out
          • 브라우저 쿠키 Opt-out :
            • 해당 브라우저에 Opt-out쿠키를 저장하며, 이후 해당 브라우저에서는 더이상 쿠키를 설정하지 않습니다. 이로 인해 Branch의 딩링킹들 서비스가 정상적으로 동작하지 않을수 있습니다.
            • 해당 브라우저만 Opt-out되는 것이며, 해당 기기 자체가 Opt-out 되는것은 아닙니다.

          • 기기 기반 Opt-out :

            • 해당 기기의 광고아이디를 포함한 양식을 제출해주시면 됩니다.

            광고주 또는 매체측의 테스트용 기기에 대해서 Opt-out하시면 Branch서비스가 제대로 동작하지 않거나 기기로그가 기록되지 않으니 유의해 주시기 바랍니다.

    Postback 및 WebHook설정¶

    • 포스트백

      • Branch 시스템에 기록된 데이터를 광고매체로 실시간으로 정보를 전달합니다.
      • [CHANNELS & LINKS] - [Ads] - [Partner Management] 페이지에서 매체별로 설정합니다.

    • 웹훅

      • 기술적으로는 포스트백과 동일하나 Branch 시스템에 기록된 데이터를 사내 데이터 시스템으로 실시간으로 전달하기 위해서 설정합니다.
      • [SETUP & TESTING] - [Data Import & Export] - [Data Feeds] 페이지에서 설정합니다.

      Facebook, Twitter와 같은 일부 SAN의 경우 웹훅을 통해서만 매체 및 캠페인 정보가 포함된 기기레벨 데이터를 전달할 수 있습니다.

    Branch 데이터 익스포트¶

    • 자세한 설명은 Data Feeds를 참고하기시 바랍니다.
    • Branch 시스템은 딥링킹 플랫폼이면서 다양한 채널로부터의 사용자 유입및 이후 행동을 측정하고 분석을 도와주는 플랫폼 입니다.
    • DataFeeds를 통해서 Branch 시스템내에 있는 데이터를 API를 통해서 익스포트 하실 수 있습니다.
    • Branch에서는 고객사에서 데이터를 효과적으로 익스포트 하실 수 있도록 가이드및 지원을 드리고 있습니다.

    • Facebook, Twitter 등의 일부 SAN(Self Attributing Network)의 경우는 해당 매체의 관련 규약에 동의하신 이후에만 해당 매체의 기기레벨의 애트리뷰션 데이터를 익스포트 하실수 있습니다. 대규모 캠페인 집행 이전에 소규모 테스트 캠페인을 통해서 기기레벨 로그에서 Facebook 애트리뷰션 정보가 정상적으로 설정되는지 사전에 확인하시는 것이 권장됩니다.

      • Facebook: https://docs.branch.io/deep-linked-ads/facebook-ads-overview/#sharing-facebook-data

      Branch는 내부 정책, GDPR 및 각종 데이터 보호 관련 법률을 준수하기 위해서 광고아이디, IP등의 정보들은 7일 이후에 자동 해슁하고 있습니다. 그러므로 해슁이전의 원본데이터를 내부적으로 장기분석의 용도로 사용하시기 위해서는 7일이내에 데이터를 익스포트 하시기 바랍니다. 일부 고객사의 경우 요청시 최대 60일까지 해슁을 지연시킬 수 있습니다.

      DataFeeds는 별도 구매하셔야하는 유료상품입니다.
