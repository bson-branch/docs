    브랜치 온보딩 가이드 (드래프트)¶

    담당자별 업무소개¶

    • Account Executive: 신규 고객에 대한 브랜치 제품 세일즈 및 계약을 담당 합니다.
    • Sales Engineer: 계약 이전 단계에서 제품의 기술적인 부분에 대한 지원을 담당합니다.
    • Client Success Manager: 계약 후 대쉬보드 사용 및 마케팅 집행에 대한 전반적인 가이드를 드립니다.
    • Solutions Engineer: 계약 이후 단계에서 Branch 연동에 관련된 기술 지원을 담당합니다.
    • Support Engineer: Branch 제품에 대한 기술,이슈등 모든 문의에 대해서 답변을 드립니다. 모든 문의사항는 support@branch.io로 메일을 보내 주시기 바랍니다.

    온보딩 작업 순서¶

    • Branch Link용 도메인 생성

      • Branch Link는 앱별로 고유한 도메인을 사용합니다.

      • app.link도메인 사용하기

        • 기본적으로 myapp.app.link와 같은 도메인을 사용하실수 있습니다.

        • 대쉬보드에서 앱 등록과정에서 미사용중인 도메인중에서 지정하실 수 있습니다.

          앱등록 이전에 미리 사용할 도메인을 정해 두시길 바랍니다.

      • 회사의 서브도메인 사용하기

        • 사용자가 문자,SNS등으로 링크를 받았을때 해당 링크를 신뢰하고 클릭할 수 있도록 myapp.mycompany.com 과 같이 회사의 도메인의 서브도메인을 사용하실 수 있습니다.
        • 회사IT팀에서 해당 서브도메인을 CNAME레코드를 custom.bnc.lt로 설정 해주시면 됩니다.
        • 자세한 사항은 아래의 문서를 참고해 주시기 바랍니다.
          • Link Settings : Advanced Settings Configuration

    • Persona 임포트

      • Branch를 사용하시기 이전에 이미 앱이 마켓에 릴리즈되어 이미 많은 인스톨이 발생되어 있는 경우 Persona임포트를 통해서 해당 사용자들를 시스템에 임포트 하는 것이며, 이를 통해서 Branch SDK가 연동된 앱이 릴리즈될때 기존사용자들에 대해서는 인스톨이 발생하지 않게 됩니다.
      • Persona임포트를 진행하시기 위해서는 CSM 또는 서포트팀(support@branch.io)으로 연락주시기 바랍니다.
      • 자세한 사항은 아래의 문서를 참고해주시기 바랍니다.
        • Importing Historical User Data
