    로그 익스포트 스크립트 가이드¶

    최종 수정일 : 2019년 4월 15일

    개요¶

    • 이 스크립트는 TUNE과 Branch시스템에서 익스포트 하는 스크립트입니다.
    • 이 스크립트는 Python v3.7.x에서 실행되며, v2.x에서는 실행되지 않습니다.
    • 현재는 한번의 요청당 최대 2백만개의 레코드까지만 익스포트가 가능합니다. 그러므로 익스포트시 레코드 갯수가 2백만개 라면, 더 많은 레코드가 있으나 일부 레코드가 익스포트되지 못함을 의미합니다. 볼륨이 많은 특정이벤트나 오픈로그의 이런 경우가 있을 수 있으며, 이 경우 시간별 또는 이벤트별로 나누어서 익스포트 하시기 바랍니다.

    1. 파이썬 설치¶

    • 아래의 웹사이트에서 Python 3.7.x을 설치합니다. Python 2.x와는 호환되지 않습니다.

      • https://www.python.org
      • https://www.python.org/downloads/

    • 맥: 특이사항 없음.

    • 윈도우: 설치시 주의사항

      • 설치 인스톨러의 화면하단 Add Python 3.7. to PATH 체크

    2. Python 설치확인¶

    • 맥: 맥에서는 2.x가 기본적으로 설치되어 있으며, 여기에 3.x를 추가로 설치하게 됩니다.
      • python 명령어로는 2.x가 사용되며, python3 명령으로 3.x버전을 사용할 수 있습니다.
      • 아래와 같이 명령어가 수행되면 정상입니다. 
        Bo-hyungs-MacBook:log son$ python --version
Python 2.7.10

Bo-hyungs-MacBook:~ son$ python3 --version
Python 3.7.3
    • 윈도우: Python이 기본으로 설치되어있지 않으며, 3.7을 설치하게 되면 python명령어로 Python 3.x를 사용하게 됩니다.
      • 아래와 같이 명령어가 수행되면 정상입니다. 
        C:\Users\Bo>python --version
Python 3.7.3

    3. PIP 설치확인¶

    • PIP는 Python에서 패키지(라이브러리)를 설치해주는 툴입니다. PIP가 정상적으로 실행되는지 확인합니다.
    • 맥: python3 명령어 사용.
      • 아래와 같이 명령어가 수행되면 정상입니다. 
        Bo-hyungs-MacBook:~ son$ pip3 --version
pip 19.0.3 from /Library/Frameworks/Python.framework/Versions/3.7/lib/python3.7/site-packages/pip (python 3.7)
    • 윈도우: python 명령어 사용.
      • 아래와 같이 명령어가 수행되면 정상입니다. 
        C:\Users\Bo>pip --version
pip 19.0.3 from c:\users\bo\appdata\local\programs\python\python37-32\lib\site-packages\pip (python 3.7)

    4. requests 패키지(라이브러리) 설치¶

    • requests는 로그 익스포트에서 사용되는 추가 설치가 필요한 패키지(라이브러리)이므로 스크립트 실행전 인스톨 하셔야합니다.

    • 맥: pip3 명령어 사용.

      • 아래와 같이 명령어를 수행했을때 설치메시지가 나오면 정상입니다. 
        Bo-hyungs-MacBook:log son$ pip3 install requests
    • 윈도우: pip 명령어 사용.
      • 아래와 같이 명령어를 수행했을때 설치메시지가 나오면 정상입니다. 
        C:\Users\Bo\log>pip install requests

    5. 스크립트 실행¶

    • 스크립트 실행 이전에 아래의 설정파일에 대한 설명을 먼저 참고하세요.
    • 스크립트는 새폴더 생성 후 특정 디렉토리에 아래의 파일들을 저장합니다.
      • 실행에 필요한 파일목록 : export.py, config.json, fields_install.csv, fields_event.csv, fields_open.csv
    • 스크립트가 저장된 폴더로 이동합니다.
      • 맥 : terminal명령어로 터미널 프로그램을 실행한 후 cd 명령어로 해당 폴더로 이동합니다.
      • 윈도우 : cmd명령어로 명령프롬프트를 프로그램을 실행한 후 cd 명령어로 해당 폴더로 이동합니다.

    • 맥: python3 명령어 사용.

      • 기본설정으로 실행
        • 아래와 같이 추가 파라메터가 없는경우 스크립트와 같은 디렉토리의 config.json 파일을 사용해서 install로그를 익스포트 합니다.
        • 아래 두가지 명령은 실제 동일합니다. 
          Bo-hyungs-MacBook:log son$ python3 export.py
Bo-hyungs-MacBook:log son$ python3 export.py --config config.json --log install

      • 설정파일을 명시적으로 지정

        • 아래와 같이 --config 파라메터 뒤에 파일명을 지정하면 특정 설정파일을 사용해서 실행합니다. 
          Bo-hyungs-MacBook:log son$ python3 export.py --config config2.json

      • 익스포트할 로그를 지정

        • 아래와 같이 --log 파라메터 뒤에 로그를 지정하면 해당 로그를 익스포트 합니다.
        • 아래는 install, event, open 로그를 각각 익스포트 하는 3가지 예시 입니다. 
          Bo-hyungs-MacBook:log son$ python3 export.py --log install
Bo-hyungs-MacBook:log son$ python3 export.py --log event
Bo-hyungs-MacBook:log son$ python3 export.py --log open

    • 윈도우: python 명령어 사용.

      • 기본설정으로 실행
        • 아래와 같이 추가 파라메터가 없는경우 스크립트와 같은 디렉토리의 config.json 파일을 사용해서 install로그를 익스포트 합니다.
        • 아래 두가지 명령은 실제 동일합니다. 
          C:\Users\Bo\log>python export.py
C:\Users\Bo\log>python export.py --config config.json --log install

      • 설정파일을 명시적으로 지정

        • 아래와 같이 --config 파라메터 뒤에 파일명을 지정하면 특정 설정파일을 사용해서 실행합니다. 
          C:\Users\Bo\log>python export.py --config config2.json

      • 익스포트할 로그를 지정

        • 아래와 같이 --log 파라메터 뒤에 로그를 지정하면 해당 로그를 익스포트 합니다.
        • 아래는 install, event, open 로그를 각각 익스포트 하는 3가지 예시 입니다. 
          C:\Users\Bo\log>python export.py --log install
C:\Users\Bo\log>python export.py --log event
C:\Users\Bo\log>python export.py --log open

    6. 스크립트 설정파일¶

    • 스크립트 : export.py

    • 설정파일 (config.json) 값 설정 :

      • 샘플 : JSON 형식입니다.

        • 스크립트 수행중 에러가 난다면 아마 잘못된 JSON 형식이기 때문일 것입니다.
        • 콤마를 주의하시고 모든 값들은 쌍따옴표로 감싸시기 바랍니다. 
              {
      "advertiser.name": "HelloWorld",
      "advertiser_id": "22012",
      "api_key" : "YOUR_API_KEY",
      "start_date": "2019-04-10",
      "days": "",
      "hours": "",
      "timezone": "Asia/Seoul",
      "filter_site_ids": ["1111", "222222"],
      "filter_site_event_names": ["purchase"],
      "branch_redirect": ""
    }

      • advertiser.name (옵션) : 광고주 이름을 설정합니다.

      • advertiser_id (필수) : 광고주 아이디를 설정합니다.
        • TUNE 대쉬보드의 [ACCOUNTS]-[Advertiser Account]-[Details] 의 Advertiser ID에서 확인하실 수 있습니다.
      • api_key (필수) :
        • TUNE 대쉬보드의 [ACCOUNTS]-[Users]에서 사용자를 선택한 후 API Keys탭에서 확인합니다.
        • 계정이 마이그레이션 되기 이전에 이미 생성된 API Key여야 양측 시스템 모두에서 동작합니다. 신규로 생성한것을 설정 하시면 TUNE에서만 동작합니다.
        • 키를 신규로 생성하시고 Branch에서도 동작하게 하시려면 서포트팀(support@branch.io)을 통해서 요청해주시기 바랍니다.
      • start_date (필수) : 년월일 YYYY-MM-DD형식 또는 년월일시분초 YYYY-MM-DDTHH-MM-SS형식
        • 시간대 별로 익스포트 할 경우에는 일시가 모두 포함된 YYYY-MM-DDTHH-MM-SS형식으로 하시면됩니다. 일단위 익스포트시에는 년월일 형식을 사용하시면됩니다.
        • 예) 2019-04-15 : 2019년 4월 15일 0시 0분 0초
        • 예) 2019-04-05T00:00:00: 2019년 4월 5일 0시 0분 0초
        • 예) 2019-04-05T01:00:00: 2019년 4월 5일 1시 0분 0초
      • days, 'hours (둘중 하나 필수) :
        • start_date로부터 어느 기간동안의 로그를 익스포트 할지 설정하며, 이값으로 end_data가 설정됩니다.
        • days와 hours 둘중 하나만 설정하셔야 하며, 둘다 설정될 경우 days값이 사용됩니다.
        • 값을 설정하지 않으려는 경우 "days": "", "hours": ""과 같이 설정하시면됩니다.
      • timezone (필수) :
        • 타임존 데이터 베이스 페이지의 TZ database name 컬럼의 값을 설정합니다.
        • 한국표준시는 Asia/Seoul, UTC는 UTC를 설정합니다.
      • filter_site_ids (필수) :
        • 특정 앱의 로그만 익스포트 할경우 설정하며, 설정하지 않으면 모든 앱의 로그가 익스포트 됩니다.
        • 필터링 하지 않을 경우 "filter_site_ids": []와 같이 설정합니다.
        • 필터링이 필요한 경우 : "filter_site_ids": ["111111", "222222"]와 같이 쌍따옴표(") 안에 사이트 아이디를 콤마(,)를 구분자로 설정합니다.
      • filter_site_event_names (필수) :
        • 이벤트로그에서만 사용되며, 특정 이벤트의 로그만 익스포트 할경우 설정하며, 설정하지 않으면 모든 이벤트의 로그가 익스포트 됩니다.
        • 필터링 하지 않을 경우 "filter_site_event_names": []와 같이 설정합니다.
        • 필터링이 필요한 경우 : "filter_site_event_names": ["purchase", "login"]와 같이 쌍따옴표(") 안에 사이트 아이디를 콤마(,)를 구분자로 설정합니다.
      • branch_redirect (필수) :
        • 로그를 TUNE 시스템에서 추출할지 Branch 시스템에서 추출할지 설정합니다.
        • 마이그레이션이 완료된 경우 TUNE에서는 신규데이터가 기록되지 않으며, 4월말까지는 TUNE시스템에서 로그 익스포트가 가능합니다.
        • TUNE 시스템에서 익스포트 할 경우 "branch_redirect": ""와 같이 설정합니다.
        • Branch 시스템에서 익스포트 할 경우 "branch_redirect": "3"와 같이 설정합니다.

    7. 추출필드 설정파일¶

    • 파일: fields_install.json, fields_event.json, fields_open.json
    • 샘플 
        created
id
status
status_code

mat_id
google_aid
os_id
ios_ifa
ios_ifv
#platform_aid
      • 스크립트 실행시 3개의 파일을 모두 스크립트 파일과 동일한 디렉토리에 있어야합니다.
      • 로그별로 fields_로 시작하며 뒤에 각각의 로그이름이 붙으며, 해당 로그 익스포트시 사용됩니다.
      • 필드명은 기존에 로그익스포트시 CSV파일의 첫번째 열에 설정되었던 필드명을 사용하시기 바랍니다.
      • 한줄에 하나씩 익스포트할 필드를 설정하시기 바랍니다.
      • #를 필드명 앞에 추가하면 해당 필드를 임시로 제외할 수 있습니다.
      • 값이 없는 행을 추가하실 수 있습니다.
