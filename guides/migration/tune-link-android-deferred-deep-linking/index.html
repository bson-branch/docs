    TUNE Link Deferred Deep Linking 가이드 (마이그레이션 이후)¶

    Modified on: Feb 28, 2019

    개요¶

    안드로이드앱에서 TUNE Link를 사용해서 브릿지 페이지로 사용자를 랜딩 시키는 경우 구글의 디퍼드 딥링킹 기능을 이용해서 디퍼드 딥링킹을 구현하는 방법을 설명드립니다.

    iOS앱의 경우는 앱스토어에서 유사한 기능을 제공하지 않기 때문에 디퍼드 딥링킹을 하기 위해서는 Attribution Analytics의 디퍼드 딥링킹 기능을 사용하셔야 함을 알려드립니다. 아래의 문서를 함께 참고 하시기 바랍니다.

    • Deferred Deep Linking with Attribution Analytics

    • Implementing a Deferred Deep Link (SDK연동방법)

    브릿지페이지 구현 샘플¶

    • 테스트및 구현 샘플페이지

    • 샘플페이지내의 TUNE Link는 아래와 같습니다. 

      https://s9u-6.tlnk.io/serve?
          action=click&
          publisher_id=174354&
          site_id_android=2385&site_id_ios=2962&
          campaign_id_android=236537&
          campaign_id_ios=237906&
          url_android=https%3A%2F%2Fbson-branch.github.io%2Fdocs%2Fguides%2Fmigration%2Fbridge%2Fbridge.html&
          url_ios=https%3A%2F%2Fbson-branch.github.io%2Fdocs%2Fguides%2Fmigration%2Fbridge%2Fbridge.html

      • 안드로이드와 iOS모두 https://bson-branch.github.io/docs/guides/migration/bridge/bridge.html 페이지로 랜딩시키도록 되어있습니다.
      • 랜딩페이지의 URL은 URL인코딩을 한 다음 설정하는 것이 권장됩니다.
      • 랜딩시 HTTPS를 사용하는 페이지로 랜딩시키야 웹뷰내에서 랜딩시 이상없이 랜딩될 수 있습니다.
      • 안드로이드앱에서 구글플레이로 랜딩시만 해당 작업이 필요하며 iOS에서 앱스토어로 리다이렉트시에는 특별한 작업이 필요 없습니다.

    구현사항 상세설명¶

    샘플페이지내의 JavaScript 소스코드에도 설명되어있는 사항을 다시한번 요약정리 하면 아래와 같습니다.

    1. TUNE Link클릭시 브릿지페이지 랜딩시 쿼리파라메터에 referrer가 추가되되며, mat_click_id=3e4c2abeef0d74958ae4bff96be434b6-20190301-877를 URL 인코딩한 mat_click_id%3D3e4c2abeef0d74958ae4bff96be434b6-20190301-877이 설정됩니다.

      • mat_click_id는 클릭시마다 매번 고유한 값이 생성됩니다.
      • 추가 리다이렉트가 발생하는 경우 referrer를 계속 다음페이지로 전달하면서 값을 가지고 있어야 합니다.

    2. referrer에 인코딩되어 설정된 mat_click_id를 디코딩합니다.

    3. 사용할 딥링크의 쿼리파라메터에 referrer에서 디코딩한 mat_click_id를 추가해서 실제 사용할 최종 딥링크URL을 생성합니다. 위의 단계에서 디코딩한 값을 쿼리파라메터에 추가하면됩니다. 단, 이미 쿼리파라메터가 있다면, & 뒤에 추가하고, 없다면 ?뒤에 추가합니다.

      • 딥링크URL 원본: 
        hellochatty://hellochatty.co/chat
      • 딥링크URL 최종: 
        hellochatty://hellochatty.co/chat?mat_click_id=2e7ea400184b74689770fc371b538254-20190301-877

    4. 구글 플레이 랜딩 URL에 브릿지페이지 랜딩시 전달 받은 referrer를 그대로 추가합니다.

      • 원본 URL: 
        https://play.google.com/store/apps/details?id=com.hellochatty
      • 인스톨 애트리뷰션용 URL : 
        https://play.google.com/store/apps/details?id=com.hellochatty&referrer=mat_click_id%3D2e7ea400184b74689770fc371b538254-20190301-877
    5. 그다음 3번에서 생성한 최종딥링크를 한번 URL인코딩 한 후 url파라메터에 설정합니다.
      • 인스톨 애트리뷰션 및 디퍼드 딥링킹용 : 
        https://play.google.com/store/apps/details?id=com.hellochatty&referrer=mat_click_id%3D2e7ea400184b74689770fc371b538254-20190301-877&url=hellochatty%3A%2F%2Fhellochatty.co%2Fchat%3Fmat_click_id%3D2e7ea400184b74689770fc371b538254-20190301-877
