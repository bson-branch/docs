    Custom Export API Guide¶

    Last updated: Sep. 18th, 2019

    Overview¶

    • 기존에 TUNE에서 로그익스포트를 위해서 제공 되었던 Reporting API를 최소한의 변경으로 Branch에서도 동일하게 사용하실수 있도록 새롭게 개발된 API 입니다.
    • 영문 공식 가이드 문서를 함께 참고해 주시기 바랍니다.

    TUNE SDK에서 Branch SDK로 변경시 주의사항¶

    • site_id, site_name:
      • TUNE에서 제공되던 OS별로 각각 앱에 부여된 아이디와 이름입니다.
      • Branch SDK를 통해서 기록된 이벤트에는 이 값들이 설정되지 않습니다.
      • Branch에서는 OS에 상관없이 앱에 부여되는 아이디와 이름이 있으며 app_id, app_name필드로 익스포트가 가능하며, 이 두 필드에 대한 추가 익스포트가 권장됩니다.
        • 대쉬보드의 Account Setting페이지에서 Branch앱의 아이디와 네임의 확인이 가능합니다.
        • 브랜치앱에서는 OS별로 각각 별도로 아이디와 네임이 부여되지 않습니다.
        • OS 구분은 sdk를 사용하실 수 있습니다.
    • site_event.name, site_event_id:
      • TUNE에서 제공되던 이벤트별로 각각 부여된 아이디와 이름입니다.
      • Branch SDK를 통해서 기록된 이벤트에는 이 값들이 설정되지 않습니다.
      • 브랜치에서 별도로 이벤트에 부여되는 name 필드에 대한 추가 익스포트하셔서 site_event.name 대신 사용하시는것을 권장합니다.

    Branch 이벤트 카테고리¶

    • 이벤트 카테고리는 Branch에서 이벤트들을 논리적으로 분류한 것 입니다.

    • Custom Export API로 로그익스포트시 datasource 필드로 해당 이벤트가 소속된 이벤트 카테고리를 알수 있음.

    • (구)TUNE에서의 논리적 이벤트 구분

      • Click, Install, Open, Update, Event 로그로 구분되어 각각의 해당 로그에서 익스포트.
      • 인스톨이후 발생되는 이벤트(Post-install Event)는 모두 하나의 이벤트로그에서 익스포트 되며, 이벤트타입(site_event.type필드)으로 논리적 구분.
      • 표준이벤트들은 이벤트타입에 해당 이벤트타입 이름이 자동 설정되며, 비표준 이벤트들은 event라는 이벤트타입 이름이 설정됨.
    • Branch에서의 논리적 이벤트 구분
      • Click, Install, Open, Reinstall, User LifeCycle Event, Content Event, Commerce Event, Custom Event 로그로 분리되어 각각의 로그에서 익스포트.
      • 인스톨이후 발생되는 이벤트(Post-install Event)들은 V2표준이벤트와 비표준이벤트의 크게 두가지로 구분되며, V2표준이벤트들은 User LifeCycle Event, Content Event, Commerce Event 로그에서 각각 익스포트 되며, 비표준이벤트들은 Custom Event로그에서 모두 익스포트됨.
    • LiveView에서 로그 확인시 이 카테고리를 우선 선택후 해당 카테고리내의 이벤트들을 확인을 할 수 있습니다.

    Branch V2 이벤트 카테고리¶

    • Branch에서는 V2표준이벤트들을 논리적으로 구분하여 User Lifecycle Event, Content Event, Commerce Event 로 구분하며, V2표준이벤트가 아닌 이벤트들은 Custom Event로 구분합니다.
    • Branch API로 익스포트
      • 이 4개의 카테고리별 별도의 엔드포인트를 통해서 각각 익스포트 합니다.
      • 이벤트 카테고리별로 데이터베이스내에 각각 별도 테이블에 저장된다고 이해하시면 됩니다.
    • Custom Export API로 익스포트
      • 실제로는 4개의 이벤트 카테고리별로 각각 별도로 저장되는 이벤트들이 하나의 events 엔드포인트를 통해서 모두 익스포트됩니다.
      • 이것은 (구)TUNE과의 호환성을 제공하기 위함입니다.
